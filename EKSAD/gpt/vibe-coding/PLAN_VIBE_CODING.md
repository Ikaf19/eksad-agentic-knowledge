## 🧭 What Is This?

This plan governs the creation of **EKSAD Vibe Coding Guides** — a third usage tier for EKSAD AI assistants, alongside:

| Tier | Platform | Setup |
|------|----------|-------|
| 1 | ChatGPT Custom GPT | Per-role `GPT_*_SETUP_GUIDE.md` files listed in `gpt/README.md` |
| 2 | Claude Project / Free Tier | `gpt/CLAUDE_SETUP_GUIDE.md` |
| **3** | **IDE Vibe Coding** ← new | **`gpt/vibe-coding/VIBE_CODING_SETUP_GUIDE.md`** |

**Vibe Coding Tier** means: AI code assist embedded directly inside the developer's IDE/editor. The AI reads the codebase, context files, and standards as it writes code — no chat window needed.

---

## 🎯 Goals

1. Each EKSAD developer/TL has a **ready-made config file** they can drop into any project repo to immediately get AI code assist that respects EKSAD standards.
2. The `_base/` knowledge files remain the **single source of truth** — vibe coding configs reference them by relative path, never duplicate them.
3. Supported tools: **GitHub Copilot** (VS Code / JetBrains), **Cursor**, **Claude Code**.
4. Phased delivery: BE Developer → FE Developer → Technical Leader.

### How Phase 1 Files Were Structured (Follow This Pattern)

**COPILOT file** — self-contained, ~150 lines:
- Header: tool + role + source file + date
- Section: Identity (who the AI is)
- Section: Context Files (list with paths, since Copilot can't auto-read)
- Section: Mandatory Patterns (full inline — Copilot can't reference external files)
- Section: Forbidden Patterns (table format)
- Section: Output Rules

**CURSOR file** — lean, ~80 lines:
- Header: tool + role
- `@file` references at top to trigger context auto-attach
- Identity block (3–5 lines)
- Mandatory patterns in condensed bullet form
- Forbidden table
- Output rules

**CLAUDE CODE file** — medium, ~100 lines:
- Header: tool + role
- `Read()` instruction block listing exact relative paths
- Identity block
- Mandatory patterns (same condensed form as Cursor)
- Forbidden table
- Output rules
- End: "After reading the files above, confirm readiness with one sentence."
