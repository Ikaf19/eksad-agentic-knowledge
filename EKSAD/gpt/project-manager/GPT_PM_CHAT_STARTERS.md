# EKSAD Project Manager — Chat Starters

## Initiation

### Create a Project Charter

```text
Bantu saya membuat Project Charter untuk [nama project]. Gunakan hanya fakta yang saya berikan. Tandai data yang belum tersedia sebagai TBD dengan owner dan due date, lalu berhenti pada status In Review.
```

### Review Charter Readiness

```text
Review Project Charter berikut terhadap EKSAD Project Management Standard. Pisahkan blocker approval, gap non-blocking, assumption, dan keputusan yang dibutuhkan.
```

## Planning

### Build a Project Plan

```text
Buat Project Plan berdasarkan Charter dan milestone commitment berikut. Jangan membuat tanggal atau resource estimate baru. Tampilkan dependency dan stage gates.
```

### Create RACI

```text
Buat RACI untuk deliverable berikut. Pastikan tepat satu Accountable per deliverable dan PM tidak mengambil accountability specialist.
```

## RAID

### Build or Review RAID

```text
Susun RAID Log dari catatan berikut. Gunakan ID immutable, risk scoring 1–5, owner, due date, trigger, mitigation, contingency, dan closure evidence.
```

### Dependency Review

```text
Review dependencies berikut. Tunjukkan provider, consumer, required-by date, acceptance criteria, fallback, dan dependency yang harus dieskalasi.
```

## Monitoring and Status

### Weekly Status

```text
Buat weekly status report berdasarkan milestone, artifact, approval, dan RAID evidence berikut. Gunakan Grey jika evidence tidak cukup dan jangan mengubah baseline.
```

### Status Challenge

```text
Challenge status report ini. Temukan RAG yang tidak didukung evidence, milestone variance yang disembunyikan, ownerless action, dan keputusan overdue.
```

## Change Control

### Assess a Change

```text
Buat Change Request untuk perubahan berikut. Petakan dampak scope, schedule, resource/cost, quality, architecture, security, operations, RAID, dan artifact IDs. Jangan update baseline sebelum approval.
```

## Gates and Escalation

### Gate Readiness

```text
Assess readiness gate [BRD/FSD/TSD/Release] dari evidence berikut. Output hanya APPROVE recommendation, REVISE, ABORT, atau SKIP/WAIVE recommendation beserta gap dan residual risk. Jangan approve atas nama authority.
```

### Draft an Escalation

```text
Buat escalation note untuk blocker berikut: decision needed, impact of delay, options, recommendation, decision owner, dan due date. Hindari bahasa menyalahkan.
```

## Closure

### Project Closure Review

```text
Review project closure readiness. Verifikasi accepted/waived deliverables, residual RAID transfer, unresolved follow-up, final approval evidence, dan lessons learned.
```
