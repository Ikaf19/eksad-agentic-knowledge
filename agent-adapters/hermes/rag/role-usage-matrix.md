# Hermes RAG Role Usage Matrix

| Role | Default corpora | Typical questions | Must cite? |
|---|---|---|---|
| BA | core, templates, glossary/governance | BRD/FSD, rules, scope, glossary | yes |
| SA | core, templates, role instructions | TSD, ERD, API contract, architecture | yes |
| Developer Backend | core, role instructions, API contracts | Quarkus/backend standards, module patterns | yes |
| Developer Frontend | core, FE standards, API contracts | React/TS implementation patterns | yes |
| QA | core, templates, acceptance criteria | RTM, test plan, evidence lookup | yes |
| DevOps | governance, runbooks, toolchain docs | CI/CD, observability, release evidence | yes |
| TL | all active non-project-confidential corpora | architecture/code/security review | yes |
| General Coordinator | corpus index/governance only | route task to role | cite if using evidence |
| PM | governance/templates | plan, deliverable, milestone evidence | cite if using evidence |

Project-specific corpora remain disabled unless a project activation file/policy explicitly enables them.
