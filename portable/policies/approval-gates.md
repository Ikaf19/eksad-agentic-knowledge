# Approval Gates

## Default gates

| Gate | Required before |
|---|---|
| BRD readiness | FSD/TSD elaboration |
| FSD approval | Implementation planning |
| TSD approval | Code implementation |
| Data analysis source/metric sign-off | Business decisions, KPI publication, or dashboard implementation |
| ML experiment review | Model promotion, production design, or operational ML work |
| UX handoff approval | Frontend implementation or design-system publication |
| Content approval | External publishing, training release, customer-facing claims, or campaign scheduling |
| QA evidence | Release sign-off |
| PM/release gate | Deployment |
| Security review | Exception/risk acceptance |

## Rule

Agent output can recommend, draft, analyze, or summarize. Approval must remain explicit and attributable.
