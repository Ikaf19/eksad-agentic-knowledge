# Content Creation Workflow

Runtime-neutral workflow for source-grounded content briefs, release notes, user guides, knowledge-base articles, training material, and content calendars.

## Stages

1. **Brief intake** — define audience, objective, channel, tone, length, deadline, source materials, and approval owner.
2. **Evidence gathering** — collect approved source docs and citations. Mark missing evidence as `[SOURCE GAP]`.
3. **Message architecture** — define core message, supporting points, call-to-action, and exclusions.
4. **Drafting** — write in the requested language/tone, preserving source truth and avoiding unsupported claims.
5. **Review checklist** — check factual grounding, audience fit, confidentiality, brand/tone, legal/regulatory sensitivity, and publication readiness.
6. **Variant production** — produce channel variants only after the core source-grounded draft is stable.
7. **Handoff** — send approval notes to product/business/legal/brand owner as needed.

## Output standards

- Cite or list source documents for all factual claims.
- Separate draft content from approval recommendations.
- Use plain language and consistent terminology from the glossary.
- Do not publish or send externally without explicit approval.

## Stop conditions

Stop and ask when the content requires legal/regulatory claims, unreleased product promises, customer-sensitive data, pricing commitments, or brand authority not provided in the source material.
