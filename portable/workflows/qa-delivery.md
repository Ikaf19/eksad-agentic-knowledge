# Qa Delivery Workflow

## Purpose

Define the runtime-neutral EKSAD workflow for `qa-delivery`.

## Inputs

- Approved upstream deliverables where applicable.
- Relevant EKSAD standards from `EKSAD/gpt/_base/`.
- Relevant templates from `EKSAD/gpt/_template/`.
- Evidence from allowed MCP capabilities if configured.

## Steps

1. Confirm role ownership and boundaries.
2. Gather required inputs and evidence.
3. Produce draft output using EKSAD templates/standards.
4. Validate traceability and quality gates.
5. Produce handoff notes for downstream roles.

## MCP rule

Use MCP only if configured, allowed for the role, and useful for evidence. Always provide a non-MCP fallback.

## Output

- Markdown deliverable or review report.
- Traceability section.
- Open questions and risks.
- Next-role handoff.
